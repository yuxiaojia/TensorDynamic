# Tensor Injector — HMMA Fault Injection Tool

HMMA Tensor Core fault injection tool for NVBit. Loads injection targets generated by `tensor_profile` and injects faults into specific HMMA instructions.

## Overview

`tensor_injector` performs targeted fault injection into CUDA Tensor Core operations. It enables controlled fault injection experiments by:

- **Target-Based Injection**: Loads pre-generated injection targets from `tensor_profile` and injects at precise locations
- **Selective Instrumentation**: Only instruments targeted kernels for minimal overhead
- **Instruction-Level Control**: Injects faults into specific HMMA register outputs at exact block/thread/instruction coordinates
- **Per-Thread Injection**: Injects faults into specific thread executions with configurable multipliers

## Key Features

- **Two-Pass Operation**: Separates profiling (tensor_profile) from injection for precise targeting
- **Runtime Configuration**: All parameters configurable via environment variables
- **Selective Kernel Injection**: Only instruments kernels present in the target file
- **Instruction Position Control**: Targets exact HMMA instruction indices per kernel
- **Thread Limiting**: Injects only at pre-specified thread locations — no duplicate injections
- **FP16 Corruption**: Separate multipliers for low and high FP16 values in each register

## Building

```bash
cd tensor_injector/
make ARCH=sm_90
```

Adjust `ARCH` based on your GPU (e.g., `sm_80` for Ampere).

## Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `INJECTION_MODE` | 0 | Injection mode: 0=single bit-flip, 1=error multiplication |
| `BIT_POSITION` | 0 | Bit position for single bit-flip mode (0-31) |
| `TOOL_VERBOSE` | 0 | Enable verbose output (0 or 1) |
| `MANGLED_NAMES` | 1 | Print kernel names mangled (1) or demangled (0) |

## Example

```bash
INJECTION_MODE=0 BIT_POSITION=5 \
LD_PRELOAD=tensor_injector/tensor_injector.so \
python3 your_eval_script.py
```
